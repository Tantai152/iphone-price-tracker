# ML Specification & Data Schema

## 1. ML Specification
This document defines the technical specifications for the Machine Learning model used for iPhone price forecasting and anomaly detection.

| Parameter          | Specification                      | Description / Rationale          |
|--------------------|------------------------------------|----------------------------------|
| Input Features     | `price_vnd` (lag 1,3,7),           | Core features to capture price   |
|                    | `day_of_week`, `is_weekend`,       | momentum, seasonality, and       |
|                    | `days_since_release`,              | source-specific behaviors.       |
|                    | `discount_pct`, `source_id`        |                                  |
| Target Variable    | `price_vnd` at t+1 to t+7          | 7-day price forecast.            |
| Main Model         | Facebook Prophet + ARIMA (baseline)| Prophet handles seasonality well;|
|                    |                                    | ARIMA serves as a standard       |
|                    |                                    | baseline comparison.             |
| Anomaly Method     | Z-score (\|z\| > 3) + IQR          | Simple, robust, and highly       |
|                    |                                    | interpretable methods for        |
|                    |                                    | outlier detection.               |
| Retrain Schedule   | Every 7 days                       | Automated via GitHub Actions.    |
| Evaluation Metric  | MAE and MAPE                       | Mean Absolute Percentage Error is|
|                    |                                    | easy to explain to non-technical |
|                    |                                    | stakeholders.                    |
| Output             | `ml_predictions` table             | Saved to the Data Warehouse and  |
|                    |                                    | exposed via an API endpoint for  |
|                    |                                    | the BI dashboard.                |

## 2. Schema Output ML
Defines the structure of the output tables generated by the Data Scientist, which will be utilized by the Data Analyst for the BI Dashboard.

### Table: `ml_predictions`

| Column Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `prediction_id` | BIGSERIAL | Primary Key | Unique identifier for each prediction record. |
| `model_id`      | INT         | Foreign Key | Links to the `dim_model` table.             |
| `source_id`     | INT         | Foreign Key | Data source used for training the model.    |
| `forecast_date` | DATE        | Not Null    | The target forecasted date (next 7 days).   |
| `predicted_price` | BIGINT    | Not Null    | Forecasted price in VND.                    |
| `lower_bound`   | BIGINT      | Nullable    | Lower confidence interval (80%).            |
| `upper_bound`   | BIGINT      | Nullable    | Upper confidence interval (80%).            |
| `model_version` | VARCHAR(50) | Not Null    | Model version used (e.g., "prophet_v1.2").  |
| `trained_at`    | TIMESTAMPTZ | Not Null    | Timestamp when the model was trained/updated.|

### Table: `ml_anomalies`
> **Note:** This table will be designed based on the `is_anomaly` flag integrated directly into the `fact_prices` table.
